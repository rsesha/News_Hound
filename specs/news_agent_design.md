# News Agent Design & Inner Workings

## 1. Architectural Overview
The News Agent is a high-performance AI search agent designed to orchestrate multiple search engines and provide synthesized, accurate answers with zero hallucinations. It follows a modular architecture split between a React/Vite frontend and a FastAPI/LangGraph backend.

*   **Frontend**: A modern React/Vite/TypeScript interface that uses WebSockets for real-time streaming of research progress (generating queries, searching, reflecting).
*   **Backend (Python/FastAPI)**: The core research engine that uses LangGraph to manage the research workflow as a state machine.
*   **LLM Layer**: Supports both Google Gemini API and local models (via Llama-Swap or Ollama) for query generation, reflection, and final answer synthesis.

## 2. The Research Pipeline (LangGraph Workflow)
The research process is managed by a state machine (defined in `backend/research_engine/graph.py`) with the following stages:

1.  **Query Generation**: The LLM takes the user's initial question and breaks it down into multiple (default 3) targeted sub-queries.
2.  **Parallel Web Search**: Each generated query is searched simultaneously across three specialized engines:
    *   **BrightData**: Social media (X, TikTok) and video content (YouTube transcripts).
    *   **Tavily**: High-density research data tailored for AI agents.
    *   **DuckDuckGo**: Standard web results without API rate limits.
3.  **Content Extraction (The Scraper)**:
    *   For every search result, `scraper.py` is used to fetch the actual page content.
    *   It removes "noise" (ads, navigation, scripts) and extracts key text (capped at 8k chars) and images.
    *   YouTube metadata and thumbnails are handled via oEmbed.
4.  **Reflection & Iteration**:
    *   The agent analyzes the gathered information for knowledge gaps.
    *   If information is insufficient, it generates follow-up queries and loops back to the search node.
5.  **Final Synthesis & Citations**:
    *   The LLM synthesizes all gathered snippets into a coherent, cited answer.
    *   The system maps internal short URLs to full source URLs in the final output to ensure transparency.

## 4. Effort Levels (Research Intensity)
The agent offers three configurable "effort" levels:

*   **Low**: 1 initial query, 1 research loop (Fast, good for simple facts).
*   **Medium**: 3 initial queries, 3 research loops (Balanced for news/summaries).
*   **High**: 5 initial queries, up to 10 research loops (Deep research for complex topics).

## 5. Persistence and Outputs
After each session, the agent generates:
*   `results.txt`: The final synthesized report with citations.
*   `search_results.tsv`: Raw search metrics and data for all engines.
*   `search_text.md`: (Optional) The full scraped content of all searched pages.
